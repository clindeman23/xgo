Step 01 — Python Basics

Let's learn some Python!

Don't worry — you don't need to know everything about Python. You just need to understand a few key ideas. Read through these and then you'll get to try writing your own code.

💬 Concept 1: Comments

A comment starts with a # symbol. Python completely ignores anything after a #. Comments are just notes for humans to read — use them to explain what your code does.

python — comments
# This is a comment — Python ignores this completely
# Use comments to explain what each part of your code does

xgo.execution_action(xgo.action_enum.SIT_DOWN)
# ↑ This line makes the dog sit

📦 Concept 2: Functions — like a math formula

Think about the slope formula: y = mx + b. A math teacher only has to write that formula out once on the board. After that, she can just say "use the slope formula" and everyone knows exactly what she means — she doesn't have to write it all out again.

A Python function works the exact same way. You define it once with all the steps written out. After that, Python knows it by name forever. You never have to write those steps out again — you just use the name.

python — defining a function (like writing the formula once)
# Just like writing y = mx + b on the board once...
# ...define the function ONCE and Python knows it forever

def do_a_trick():
    # Everything indented here is the "recipe" inside the function
    xgo.execution_action(xgo.action_enum.WAVE)
    xgo.execution_action(xgo.action_enum.SIT_DOWN)

# Now anywhere in your code, just say do_a_trick()
# Python knows exactly what that means!
🔑

def keyword

def stands for "define." It tells Python "I'm about to teach you a new function." Always followed by the name and ():

↵

Indentation matters!

Everything inside a function must be indented — press Tab or use 4 spaces. Python uses spacing to know what's "inside" the function.

🐾 Concept 3: Dot notation — reading it left to right

You'll see code like xgo.execution_action(xgo.action_enum.SIT_DOWN) and it might look scary at first. Let's break it down. The dot ( . ) just means "look inside this and use..." — like a file path on a computer.

xgo.execution_action( xgo.action_enum.SIT_DOWN )
xgo = the dog's remote control
execution_action = the command type
action_enum = the menu of actions
SIT_DOWN = the specific action to do
💜 Why does it say "action_enum"? What's an enum?

Great question! An enum is short for "enumeration" — which just means a fixed list of named options. When you see action_enum, it's just saying "look in the official list of actions." Think of it like a menu at a restaurant — you can only order things that are on the menu. The enum is the menu, and SIT_DOWN, WAVE, WHIRL etc. are the items you can order. You can't make up your own — you have to pick from the list.

📐 Concept 4: Program structure

Every XGO program follows the same pattern. Functions go at the top. The setup lines — the ones that turn on the dog — go at the very bottom.

python — full program structure
# ── 1. Define your functions at the top ──
def on_button_a():
    xgo.execution_action(xgo.action_enum.WAVE)
input.on_button_pressed(Button.A, on_button_a)

# ── 2. Setup ALWAYS goes at the very bottom ──
xgo.init_xgo_serial(SerialPin.P2, SerialPin.P1)
basic.pause(1000)
xgo.execution_action(xgo.action_enum.GO_PRONE)

🚨 Common mistakes — read this carefully!

😬

Wrong indentation

Everything inside a function must be indented. If you forget, Python throws an IndentationError.

😬

Missing the colon

def my_function() without the : at the end. That colon is required every single time.

😬

Wrong capitalization

All XGO actions are ALL CAPS — SIT_DOWN not Sit_down. Python is totally case-sensitive.

😬

Indenting on_button_pressed

This is the #1 mistake — see the big warning below!

⚠️ The #1 mistake: where to put input.on_button_pressed

The input.on_button_pressed line goes at the same level as def — NOT indented inside the function. If you indent it inside, the button will never work no matter what you try.

❌ WRONG — indented inside
def on_button_a():
    xgo.execution_action(...)
    input.on_button_pressed(
        Button.A, on_button_a)
✅ RIGHT — same level as def
def on_button_a():
    xgo.execution_action(...)
input.on_button_pressed(
    Button.A, on_button_a)

How to use movement — notice direction_enum instead of action_enum!

Speed goes from 0 (stopped) to 100 (full speed). Start at 50. To stop the dog, call DEFAULT_POSTURE.

example — movement
def walk_and_sit():
    xgo.move_xgo(xgo.direction_enum.FORWARD, 50)
    basic.pause(3000)   # walk for 3 seconds
    xgo.execution_action(xgo.action_enum.DEFAULT_POSTURE)  # stop!
    basic.pause(1000)
    xgo.execution_action(xgo.action_enum.SIT_DOWN)
input.on_button_pressed(Button.B, walk_and_sit)

xgo.init_xgo_serial(SerialPin.P2, SerialPin.P1)
basic.pause(1000)
xgo.execution_action(xgo.action_enum.GO_PRONE)

Step 05 — Setup

Setting up the real dog!

Now that you understand the code, let's get the hardware connected. Follow these steps in order and you'll be ready to go.

1

Lay the dog flat first 🐕

Before turning anything on, put the XGO in the prone position — lying flat on its belly. If you turn it on while standing upright, the joints can jam. Belly down, always!

2

Connect the cables — this is important! 🔌

There are three DuPont cables coming from the dog's body that connect to the Ring:bit board on its back. Connect them exactly like this:

⚠️ Get the cables right — it matters!

Blue → first pin of the 1vg section. This is P1 (RX).
Yellow → first pin of the 2vg section. This is P2 (TX).
Black → last pin of the 2vg section. This is ground (G).
This is exactly why our code says init_xgo_serial(SerialPin.P2, SerialPin.P1) — P2 first, P1 second.

3

Turn on the Ring:bit switch

The Ring:bit board has a small slide switch on its edge. Slide it to the ON position. This is separate from the dog's main power button — both must be on.

4

Press the XGO power button

Press the round button on the dog's spine. The ring light will glow. Give it 5 seconds to settle into position before pressing any buttons. If the ring light is flashing, the battery is low — charge it first!

5

Plug the micro:bit into your computer

Use the USB cable. Your computer will recognize it as a USB drive called MICROBIT.

⚠️ Remember: TWO switches, not one!

The dog needs two things powered on: the Ring:bit slide switch AND the XGO power button. If the dog doesn't respond to any code, check that both are on — this is almost always the reason!

Step 07 — First Code

Your first complete program!

Here's a full working program for the real dog. Button A sits it. Button B waves. Both together walks it forward. Type this into MakeCode exactly as shown, then download it!

python — your first complete program (copy this exactly!)
# ---- Button A → Sit ----
def on_button_a():
    xgo.execution_action(xgo.action_enum.SIT_DOWN)
input.on_button_pressed(Button.A, on_button_a)

# ---- Button B → Wave ----
def on_button_b():
    xgo.execution_action(xgo.action_enum.WAVE)
input.on_button_pressed(Button.B, on_button_b)

# ---- A + B → Walk forward ----
def on_button_ab():
    xgo.move_xgo(xgo.direction_enum.FORWARD, 50)
    basic.pause(2000)
    xgo.execution_action(xgo.action_enum.DEFAULT_POSTURE)
input.on_button_pressed(Button.AB, on_button_ab)

# ---- Setup — always at the very bottom ----
xgo.init_xgo_serial(SerialPin.P2, SerialPin.P1)
basic.pause(1000)
xgo.execution_action(xgo.action_enum.GO_PRONE)
⚠️ Reminder: input.on_button_pressed is NOT indented inside the function

Look at the code above carefully. Each input.on_button_pressed line is at the same level as def — not indented inside. This is the #1 mistake. If you indent it inside the function, the button will never respond.
