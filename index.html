Step 01

Getting Ready

Before you write any code, you need to connect everything and open the right tool. Follow these steps in order.

1

Lay the dog flat first

Before turning anything on, put the XGO in the prone position — lying flat on its belly. If you power it on while it's upright, the joints can jam. Belly first, always.

2

Plug the micro:bit into your computer

Use the USB cable. It shows up on your computer like a USB drive.

3

Go to makecode.microbit.org

Click New Project, name it (like "XGO Dog"), and click Create.

4

Load the XGO extension

Click Extensions in the toolbox. Search for XGO and select it. This gives MakeCode all the dog commands.

5

Switch to Python mode

At the top of MakeCode, click Python (not Blocks, not JavaScript). That's where you'll work.

⚠️ Two things that trip everyone up

1. The Ring:bit board (where the micro:bit plugs into the dog) has its own power switch — make sure it's ON.
2. After you download your code, drag the .hex file onto the micro:bit. The light will flash while it loads — wait for it to finish before testing.

Step 02

Python Basics

Never written Python before? Here's everything you actually need for this project — no fluff.

1. Comments — notes for humans

A comment starts with #. Python ignores that whole line. Use them to explain what your code does.

# This is a comment — Python ignores it
xgo.execution_action(xgo.action_enum.Sit_down)  # ← this makes the dog sit

2. Functions — name a block of code

Use def to create a function. Everything indented inside it runs when you call it. The colon at the end of def myfunction(): is required.

def do_a_trick():          # define the function
    xgo.execution_action(xgo.action_enum.Wave)  # indented = inside
    xgo.execution_action(xgo.action_enum.Sit_down)

do_a_trick()               # call it (actually runs it)
🔑

The colon :

Every def line ends with :. Forget it and Python breaks.

→

Indentation

Everything inside a function must be indented (4 spaces or Tab). Python uses this instead of curly braces.

📞

Calling vs. defining

Writing def just creates the function. Adding () at the end is what actually runs it.

3. Dot notation — how you talk to the dog

The dot . means "look inside this thing and use this tool." Think of xgo as the dog's remote control.

xgo.execution_action(xgo.action_enum.Sit_down)
# xgo = remote | execution_action = the button | Sit_down = which action

4. The program structure — always the same pattern

# 1. Initialize the dog (ALWAYS FIRST)
xgo.init_xgo_serial(SerialPin.P1, SerialPin.P2)
xgo.execution_action(xgo.action_enum.Default_posture)

# 2. Define your functions
def on_button_a():
    xgo.execution_action(xgo.action_enum.Wave)

# 3. Connect functions to buttons
input.on_button_pressed(Button.A, on_button_a)
Common mistakes

Wrong indentation — Python will throw an IndentationError. Always use 4 spaces or Tab.
Missing the colon — every def line needs : at the end.
Wrong capitalization — Sit_down and sit_down are different. Check your caps.

Step 03

What the Dog Can Do

Every program starts with the two setup lines. After that, mix and match any commands below.

Always goes first

xgo.init_xgo_serial(SerialPin.P1, SerialPin.P2)
xgo.execution_action(xgo.action_enum.Default_posture)

Actions

Use: xgo.execution_action(xgo.action_enum.NAME)

NameWhat happens
Default_postureStand up straight — use at start and to reset
Sit_downDog sits
WaveRaises and waves one front leg
Shake_handsExtends a paw forward
DanceBuilt-in dance routine
Mark_timeMarches in place
PeeLifts one back leg 😂
StretchStretches front legs forward

Movement

Use: xgo.move_xgo(xgo.direction_enum.DIRECTION, speed)

Speed: 0 = stopped, 100 = full speed. Start around 40–60.

DirectionWhat happens
ForwardWalk forward
BackwardWalk backward
Turn_leftRotate left on the spot
Turn_rightRotate right on the spot
LeftCrab-walk left
RightCrab-walk right

Timing and loops

CommandWhat it does
basic.pause(ms)Wait before the next line. 1000 = 1 second, 500 = half second
basic.forever(fn)Runs your function over and over forever

Buttons

Use: input.on_button_pressed(Button.X, function_name)

ButtonWhat it listens for
Button.ALeft button on the micro:bit
Button.BRight button on the micro:bit
Button.ABBoth pressed at the same time

Step 05

Ready to Level Up?

Try these in order. Each one builds on what you learned. Don't peek at the Try It page unless you're really stuck.

EASY

1. The Happy Dog

Button A → wave. Button B → sit. Start in Default_posture. Use functions and on_button_pressed.

EASY

2. The Trick Routine

When the program starts: stand → sit → pause 2 sec → wave → pause 1 sec → stand. Use basic.pause() between each move.

MEDIUM

3. The Loop Dog

Make the dog repeat a 3-action routine forever using basic.forever(). Add pauses between each action so it looks natural.

MEDIUM

4. Walk a Square

Forward → turn right → repeat 4 times. Experiment with timing using move_xgo() and pause() to make it actually square.

HARD

5. Full Trick Show

Button A = 3 tricks in a row. Button B = walk forward 3 seconds then stop. A+B = do something creative. Use functions, pauses, movement, and actions all together.

Bonus: Make it your own

Create something that doesn't exist in any of these challenges. Name it. Be weird. The best code comes from people who go off-script.
